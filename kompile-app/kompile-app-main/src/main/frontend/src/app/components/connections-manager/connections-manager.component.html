Setup Instructions

To enable OAuth connections, you need to configure the client credentials for each provider:

google Google

  1. Go to Google Cloud Console
  2. Create a new project or select existing
  3. Enable Drive API
  4. Create OAuth 2.0 credentials
  5. Add redirect URI: /api/oauth/google/callback
  6. Set kompile.oauth.google.client-id and kompile.oauth.google.client-secret

microsoft Microsoft

  1. Go to Azure Portal
  2. Register a new application
  3. Add API permissions (Files.Read, User.Read)
  4. Create client secret
  5. Add redirect URI: /api/oauth/microsoft/callback
  6. Set kompile.oauth.microsoft.client-id and kompile.oauth.microsoft.client-secret

link Atlassian

  1. Go to Atlassian Developer Console
  2. Create a new OAuth 2.0 app
  3. Add Confluence and Jira scopes
  4. Add redirect URI: /api/oauth/atlassian/callback
  5. Set kompile.oauth.atlassian.client-id and kompile.oauth.atlassian.client-secret

auto_stories Notion

  1. Go to Notion Integrations
  2. Create a new integration
  3. Select OAuth type
  4. Add redirect URI: /api/oauth/notion/callback
  5. Set kompile.oauth.notion.client-id and kompile.oauth.notion.client-secret

tag Slack

  1. Go to Slack App Directory
  2. Create a new app
  3. Add OAuth scopes (channels:history, channels:read)
  4. Add redirect URI: /api/oauth/slack/callback
  5. Set kompile.oauth.slack.client-id and kompile.oauth.slack.client-secret
