Why this process — and what we measured before automating it

Section 3

Cross-cutting planes — for this specific process

● Governance

  • SOX scope: Phase 4 sign-off and IC eliminations are ICFR-relevant — agent participation requires control documentation
  • FX policy: forward as of last BD of prior month (per FX Policy Memo §2) — encoded in FX agent spec
  • Audit retention: 7-year retention of evidence graph + run logs (S3 with object lock)
  • Data residency: regional sales data stays in customer cloud; agent specs reference but don't store
  • EU AI Act: variance triage classified as limited risk (HITL on every auto-fix); commentary classified limited risk (CFO edits final)

● Operating model

  • Customer sponsor: M. Sato (CFO)
  • Process steward: M. Chen (VP FP&A)
  • Ontology custodian: M. Chen + ProssimoTech CS
  • HITL approver pool: J. Park (regional triage), M. Chen (commentary), L. Okafor (close mechanics), M. Sato (projection)
  • Escalation: low-confidence triage → analyst within 4h SLA; agent failure → on-call within 1h
  • Cadence: monthly close + quarterly process review with ProssimoTech CS

● Observability / quality

  • 7 controls (C-01..C-07) fired every cycle; pass rate displayed in dashboard
  • Eval sets: golden cycles for triage agent (50 historical workbooks with known correct triage)
  • Drift monitors: triage hit rate, commentary edit-rate, FX deviation, pipeline coverage band stability
  • Replay sandbox: run new agent versions against last 6 cycles before promotion
  • Coverage metric: % of L1 evidence claims still validated by latest evidence
Section 4

Closed-loop feedback — telemetry from this process

From Layer 5 → Layer 2
↓ ontology refinement
Auto-corrected channel mismatches feed pattern library
Each accepted auto-fix in the Triage agent strengthens the relevant VarianceTriagePattern entry. Rejected fixes update the threshold rules. After 12 cycles, triage hit rate improved from 79% → 92%.
From Layer 5 → Layer 5
↓ eval set growth
CFO commentary edits become house-style training data
Every M. Chen / CFO edit on auto-drafted variance commentary becomes a held-out eval pair. The Commentary agent's spec eval set grew from 12 examples to 89 over 6 quarters.
From Layer 5 → Layer 1
↓ re-discovery trigger
ERP migration auto-triggers re-discovery
Detected at the integrations layer (NetSuite saved-search hash drift > 50%). Triggers a re-mining cycle of Layer 1 with focused scope on the changed system; ontology is patched, not rebuilt.
From Layer 3 → Layer 1
↓ continuous discovery
New SKU launch triggers narrow re-mine
When the SKU master gains an entry (e.g., HYD-110 Sleep Serum), Layer 1 spawns a narrow discovery sub-task — interview product manager, scan launch playbook, surface any new pricing or channel rules.
From Layer 5 → Layer 3
↓ process drift detection
Recurring exception class triggers process update
If the same exception class fires > 3 cycles in a row, the Process layer is flagged for review. Last quarter, this surfaced a missing rule for the new Sephora gifting channel — codified into Layer 2 ontology.
From Layer 0 → all
↓ baseline refresh
Quarterly baseline measurement
Engagement-layer metrics (analyst hours, errors caught, cycle time) are re-measured every quarter. Drift > 15% triggers a process review with the customer sponsor (M. Sato).
